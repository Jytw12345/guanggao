📋 系统概述

本系统是一款专为广告安装施工团队设计的项目预约与工时管理系统，支持本地单机模式和云端多人协作模式，采用 PWA 技术实现离线可用和桌面安装功能。

系统架构

  • 前端技术：HTML5 + CSS3 + JavaScript（原生）
  • 数据存储：浏览器 LocalStorage（本地模式）/ Supabase（云端模式）
  • 离线支持：Service Worker 缓存机制
  • 响应式设计：支持桌面端和移动端

访问帮助

在应用内点击页面右上角的 📖 帮助 按钮，可快速查看简要使用说明。

👤 用户角色

角色权限说明
总经理拥有全部权限，可管理所有项目、人员、门店和账号
店长管理本门店的项目和施工人员，可查看工时统计
施工人员查看分配给自己的项目，填写施工工时，完成维修工单

📊 项目状态流程

预约中 → 施工中 → 已完工 → 已验收 → 已审核 ↓ ↓ 已取消 发起维修单

状态说明

状态说明可执行操作
预约中项目已创建，等待施工开始施工、编辑、删除
施工中项目正在进行完成安装、查看进度
已完工安装完成，待验收确认验收、填写工时
已验收验收通过修改验收、发起维修单

📱 功能模块详解

1. 项目预约

主要功能：

  • 新建项目预约
  • 编辑/删除项目
  • 按状态、门店、时间范围筛选
  • 搜索项目名称、客户、地址
  • 超期项目红色警告标识

操作步骤：

  1. 点击「新建预约」按钮
  2. 填写项目信息：项目名称、客户名称、联系电话、门店、地址、预约时间、预计工时、施工人员分配、外协人员（可选）、备注
  3. 点击「保存」完成创建

时间筛选：

  • 全部：不限制时间范围
  • 3天/7天/15天/30天：显示对应天数内的项目

2. 日历视图

主要功能：

  • 按月查看每日预约数量
  • 显示当天工时饱和度（正常/排满/超载）
  • 点击日期查看详细安排

颜色标识：

  • 🟢 预约中
  • 🟡 施工中
  • 🟢 已完工
  • 🔵 已验收

3. 施工管理（核心模块）

3.1 施工安排描述

自动生成当天的施工安排总结，包括：

  • 项目数量统计
  • 出勤/请假/待命人员
  • 任务负荷评估（轻松/适中/紧张）
  • 每位施工人员的任务详情

3.2 项目详情管理

  • 人员分配：为项目分配施工人员
  • 工时填写：分人填写实际施工工时
  • 外协工时：单独记录外协人员工时（紫色标识）
  • 进度追踪：实时显示项目进度条

3.3 状态操作

  • 开始施工：记录开工时间，状态变为"施工中"
  • 完成安装：弹出表单记录完工时间和总用时，填写各人员工时
  • 确认验收：弹出验收表单，填写验收信息

3.4 验收表单字段

字段说明必填
验收人验收负责人姓名✓
验收时间验收日期
验收类型现场验收/远程验收/第三方验收/内部验收
验收方式实物验收/图片验收/视频验收/混合验收
验收结果合格/整改后合格/不合格
验收结论通过/有条件通过/不通过
验收项检查安装位置准确、安装牢固、外观整洁等
验收备注现场情况、遗留问题等

4. 工时统计

主要功能：

  • 按月统计施工人员工时
  • 区分内部人员和外协人员
  • 显示每日工时明细
  • 导出 CSV 报表

统计维度：

  • 总工时
  • 出勤天数
  • 参与项目数
  • 请假记录

5. 店面统计（总经理专属）

  • 按门店统计预约数量和工时
  • 工时差异分析
  • 外协人员统计
  • 导出报表

6. 施工人员管理

  • 添加/编辑施工人员信息
  • 记录人员联系方式

7. 请假管理

7.1 请假申请

  • 选择请假类型：事假、病假、年假、调休、其他
  • 设置开始/结束时间（仅计算工作时间段）
  • 自动检测与项目排期冲突
  • 提交审批

7.2 审批流程

  • 待审批列表（总经理/店长可见）
  • 批准/拒绝操作
  • 显示冲突项目提示

7.3 节假日管理

  • 添加节假日和调休日期
  • 请假计算自动排除节假日

8. 门店管理

  • 添加/编辑门店信息
  • 记录门店地址和联系方式

9. 账号管理（云端模式）

  • 查看所有注册账号
  • 分配角色权限
  • 管理门店权限

10. 角色权限配置

  • 按角色配置可执行操作
  • 权限包括：创建/编辑/删除项目、分配人员、审批等
  • 总经理恒拥有全部权限

🛠️ 特色功能

1. 人员冲突检测

  • 分配人员时自动检测时间冲突
  • 请假申请时检测与已安排项目的冲突
  • 显示冲突项目详情

2. 外协人员管理

  • 独立管理外协人员工时
  • 外协人员工时以紫色标识
  • 不参与内部人员冲突检测

3. 进度自动追踪

  • 根据状态自动计算进度百分比
  • 超期项目自动标记警告
  • 实时更新进度显示

4. 离线可用

  • Service Worker 缓存核心资源
  • 网络断开时仍可访问已缓存数据

5. 自动更新

  • 检测到新版本时提示用户刷新
  • 后台自动下载更新
  • 用户确认后应用新版本

6. 响应式设计

  • 桌面端完整功能展示
  • 移动端底部导航栏

🌐 部署说明

本地模式

无需配置，直接打开 index.html 即可使用，数据存储在浏览器本地。

云端模式（Supabase）

  1. 注册 Supabase 账号
  2. 创建新项目
  3. 在 config.js 中配置：
    • SUPABASE_URL：项目 URL
    • SUPABASE_ANON_KEY：匿名 API 密钥
  4. 在 SQL Editor 执行 supabase_schema.sql 创建数据表
  5. 执行 supabase_rbac_migration.sql 配置角色权限

GitHub Pages 部署

  1. 将项目推送到 GitHub 仓库
  2. 开启 GitHub Pages 服务

⚠️ 关键更新注意事项

自动更新机制

本系统通过 Service Worker 实现自动更新，但需要注意：

⚠️ 重要

每次更新必须修改缓存版本号：

  1. 打开 sw.js 文件
  2. 将 const CACHE = "ad-install-v3" 的版本号递增（如 v4、v5）
  3. 推送代码到 GitHub

GitHub Pages 缓存限制

  • GitHub Pages 对 sw.js 文件有约 10 分钟的 HTTP 缓存
  • 即使推送了更新，客户端可能需要等待几分钟才能检测到

更新流程

你推送更新 → 用户打开应用 → Service Worker 检查更新 → 发现新版本 → 后台安装 → 弹出提示"发现新版本！是否立即刷新？" → 用户确认 → 页面刷新 → 使用新版本

📱 PWA 安装指南

桌面端（Chrome/Edge）

  1. 打开应用后，地址栏右侧会显示安装图标
  2. 点击安装图标，确认安装
  3. 应用将添加到桌面，独立窗口运行

移动端（iOS Safari）

  1. 打开应用后，点击底部「分享」按钮
  2. 选择「添加到主屏幕」
  3. 确认添加，应用将安装到主屏

移动端（Android Chrome）

  1. 打开应用后，地址栏下方会弹出安装提示
  2. 点击「安装」按钮
  3. 应用将添加到应用列表

⚠️ 注意事项

  1. 数据备份：本地模式下数据存储在浏览器中，清除浏览器数据会导致数据丢失，建议定期导出数据
  2. 网络同步：云端模式下需要网络连接才能同步数据，离线时操作在本地缓存，联网后自动同步
  3. 更新机制：每次代码更新需修改 sw.js 中的缓存版本号，否则客户端无法检测到更新
  4. GitHub Pages 缓存：GitHub Pages 对 sw.js 有约 10 分钟的 HTTP 缓存，更新可能延迟生效
  5. 角色权限：新注册账号默认无权限，需总经理在「账号管理」中分配角色后才能使用
